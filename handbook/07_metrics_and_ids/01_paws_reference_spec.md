# PAWS Reference Specification (ID Generation)

To ensure national-level traceability and zero duplicates, every candidate dog is assigned a unique **PAWS Reference ID** at the moment of intake.

## 🆔 ID Structure: `PAWS-YY-XXXXX`

| Segment | Example | Description |
| :--- | :--- | :--- |
| **Prefix** | `PAWS` | Project Identifier. |
| **Year** | `26` | The last two digits of the intake year. |
| **Sequence** | `00042` | A 5-digit sequential number (allows for 99,999 dogs per year). |

**Example:** `PAWS-26-00402`

## 🛡️ Implementation Rules
1.  **Automated**: The ID is generated by the Supabase database engine using a PostgreSQL sequence. Human error is impossible.
2.  **Unique**: A database constraint prevents the same ID from ever being assigned to two different dogs.
3.  **Auditable**: Once an ID is generated, it is permanently linked to the dog's microchip, veterinary records, and SAPS 505(a) donation form.
4.  **Immutable**: The ID cannot be changed or "re-used" even if a dog is withdrawn from the program.

## 🔍 Public Verification
The public can verify any PAWS ID via the **[Transparency Tracker](../../docs/tracker/)**. If a dog is presented as "PAWS-Certified" but doesn't appear in the database, it is a red flag for fraud or smuggling.
